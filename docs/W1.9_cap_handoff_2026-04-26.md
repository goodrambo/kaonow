### 2.4 兩個踩坑（後續一定會再遇到）

#### Pitfall A：urllib + 中文 system prompt → `latin-1 codec error`
症狀：`'latin-1' codec can't encode characters in position 13-14`

原因：`urllib.request` 內部把 header / payload 用 latin-1 編碼，遇到非 ASCII 直接死。

**修法**：改用 `subprocess.run(["curl", ...])` + tempfile 寫 JSON body：

```python
with tempfile.NamedTemporaryFile(mode="w", suffix=".json", delete=False, encoding="utf-8") as f:
    f.write(body)
    body_file = f.name
cp = subprocess.run([
    "curl", "-sS", "-X", "POST", ANTHROPIC_API_URL,
    "-H", f"x-api-key: {api_key}",
    "-H", f"anthropic-version: {ANTHROPIC_VERSION}",
    "-H", "content-type: application/json",
    "--data-binary", f"@{body_file}",
    "--max-time", "90",
    "-w", "\n%{http_code}",
], capture_output=True, text=True, timeout=120)
```

`cap_math_vision_fix.py` 已套用。其他 vision / API call 腳本如果遇到一樣錯誤，照抄。

#### Pitfall B：PostgreSQL SQL string literal 不要 double backslash
症狀：DB 寫入 `\frac{1}{2}` 結果存成 `\\frac{1}{2}` 兩個 backslash，KaTeX 完全不渲染。

原因：PG 預設 `standard_conforming_strings=on`，SQL 字串 `'\frac'` 已經是字面 `\frac`，不需要 double。

```python
# ❌ 錯（MySQL / 舊 PG 寫法）
stem_lit = stem.replace("\\", "\\\\").replace("'", "''")
opts_lit = json.dumps(opts).replace("\\", "\\\\").replace("'", "''")

# ✅ 對（PG standard_conforming_strings=on）
stem_lit = stem.replace("'", "''").replace("\x00", "")
opts_lit = json.dumps(opts, ensure_ascii=False).replace("'", "''").replace("\x00", "")
```

**jsonb 不需要這層 double**：SQL 先 parse 字串，再交給 jsonb parse JSON。`json.dumps` 已把 `\` → `\\`（JSON spec），這時 SQL literal 直接寫不要再 double。

**驗證方法**：
```sql
SELECT id, length(question_text) AS char_len, octet_length(question_text) AS byte_len, question_text
FROM public.questions WHERE id = 'cap-104-math-001';
```
或前端 console：`(text.match(/\\/g)||[]).length` 看真實 backslash 數量。

正確範例：`cap_superscript_fix.py:341-343`。

## 7. 前端關鍵點（`index.html`）

- **`Q_CACHE_PREFIX`**（題庫 cache）：目前 `kaonow_q17_`，每次 DB 內容改要 bump
- **`loadQuestions()` 必須 paginate**：PostgREST 預設 1000 row cap，cap exam 1,645 row 會被截
  ```js
  const PAGE = 1000;
  let allRows = [];
  for (let offset = 0; offset < 20000; offset += PAGE) {
    const r = await request('questions_published?exam_id=eq.' + examId +
      '&offset=' + offset + '&limit=' + PAGE + '&order=id.asc&select=...');
    if (!r.ok || !Array.isArray(r.data)) break;
    allRows = allRows.concat(r.data);
    if (r.data.length < PAGE) break;
  }
  ```
